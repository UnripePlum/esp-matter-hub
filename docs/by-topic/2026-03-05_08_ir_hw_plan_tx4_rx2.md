# 8) IR 하드웨어/펌웨어 구현 계획 (TX 4 + RX 2)

## 목표
- IR 송신 모듈 4개를 항상 동시에 같은 신호로 송신(broadcast)
- IR 수신 모듈 2개를 독립 입력으로 받아 학습 성공률 향상
- Matter 슬롯 명령 -> 저장 신호 조회 -> 실제 IR TX까지 연결

## 핀 계획 (ESP32-S3-DevKitC-1)
- TX 제어 GPIO: `GPIO4`
- RX #1 GPIO: `GPIO5`
- RX #2 GPIO: `GPIO6`

회피 핀:
- `GPIO0` (보드 버튼/부트 스트랩)
- `GPIO38` (기본 LED 드라이버)

## 하드웨어 구성

### TX (4채널 동시 송신)
- ESP32의 `GPIO4`를 4개 TX 드라이버 입력으로 fan-out
- 각 TX 채널은 MOSFET 기반 저측 스위칭 권장
- IR LED 전류 제한 저항 필수
- TX 전류가 큰 경우 5V 별도 전원 사용, GND 공통

### RX (2채널 독립 수신)
- VS1838B #1 OUT -> `GPIO5`
- VS1838B #2 OUT -> `GPIO6`
- 두 OUT 신호는 절대 직결하지 않고 개별 GPIO로 수집

## 펌웨어 구현 방향
- `main/ir_engine.cpp`에 RMT TX/RX 실구현
  - TX: 38kHz carrier + payload duration 기반 송신
  - RX: 2채널 ringbuffer 수집 후 유효 프레임 선택
- 학습 상태 확장
  - `rx_source` (어느 수신기에서 채택했는지)
  - `captured_len` (캡처 길이)
  - `quality_score` (간단 품질 점수)
- `/api/learn/status`에 위 상태 필드 노출

## 데이터 저장
- 현재 구현은 signal table blob에 payload를 저장
- payload 길이 상한(`payload_ticks[128]`) 초과 시 잘림/실패 가능
- 차기 개선:
  - 메타데이터 테이블 + payload blob 분리 저장
  - AC 리모컨 등 장문 프레임 대응

## 검증 절차
1. TX 단독: 테스트 payload 송신 시 IR LED 4채널 동시 발광 확인
2. RX 단독: 각 수신기에 대해 학습 상태가 `ready`로 전환되는지 확인
3. E2E: 학습 -> commit -> slot bind -> Home 앱 제어 -> IR 송신 로그 확인
4. 재부팅: 저장된 signal/binding 유지 및 재송신 확인
